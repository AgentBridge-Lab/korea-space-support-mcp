# Korea Space Support — Web 검색 UI 기획서

작성일 2026-06-10 KST · 버전 v1 (초안)

---

## 1. 페르소나 & 핵심 니즈

**1차 타깃**: 대학 연구실·연구자 / 출연연 / 연구팀 리더

| 페르소나 | 행동 | 핵심 결정 정보 |
|---|---|---|
| 연구실 PI | "내 분야 + 신청 가능한 R&D 사업 빠르게 훑기" | 마감일, 신청자격(연구실/대학 OK?), 컨소시엄 필수 여부 |
| 박사후 연구원 | "내가 단독 또는 팀으로 들어갈 수 있는 신규 과제 찾기" | 연구자/연구팀 대상 여부, 분야 적합성, 보안요건 |
| 산학협력단/연구지원팀 | "교내 PI들에게 추천할 사업 모음" | 출처 명확성, 원문 링크, 마감 임박 알림 |

비차별 가치 제안: **"읽을 수 있는 마감일이 있는 공고만 보여준다."** 가이드·포털·결과공고 제거.

## 2. 정보 구조 (Information Architecture)

```
/  (메인 검색 페이지, 탭 모드 3종)
├─ ?mode=keyword   (자연어 검색 + 사이드 필터)
├─ ?mode=filter    (필터 패널 중심, 키워드는 보조)
└─ ?mode=match     (회사·연구실 프로필 → fit-score 추천)

/program/[id]       (공고 상세)
/about              (수집 정책·출처·면책)
```

상단 전역 영역: 로고 / 마지막 갱신 시각 배지 / "About 수집정책" 링크.
하단 푸터: 라이선스, 데이터 면책, GitHub 링크.

## 4. 결과 카드 (Result Card)

```
┌──────────────────────────────────────────────────────────────┐
│ ● 모집중   위성     연구실/기업 컨소시엄    방산 X         │  ← 배지 행
├──────────────────────────────────────────────────────────────┤
│ 2026년 ◯◯◯ 위성 R&D 지원사업                              │  ← 제목 (큰글)
│ 한국항공우주연구원 · KARI                                    │  ← 기관
├──────────────────────────────────────────────────────────────┤
│ 📅 신청기간  2026-06-15 ~ 2026-07-20  (D-40)                │  ← 마감
│ 💰 지원규모  과제당 최대 5억원 (원문 확인)                  │  ← 액수
│ 👥 신청자격  연구실·기업 컨소시엄 가능                       │  ← 자격 요약
├──────────────────────────────────────────────────────────────┤
│ "본 사업은 ◯◯◯ 위성 활용 기술의 핵심 부품 국산화를…"      │  ← 짧은 요약 (≤200자)
├──────────────────────────────────────────────────────────────┤
│ 원문 보기 →     원기관 출처 →     상세 정보 보기 →          │  ← 액션
└──────────────────────────────────────────────────────────────┘
```

**배지 색상 (정부·공공 톤)**:
- 모집중 → 진한 청색 (#1F4BB4)
- 마감 임박 (D-7 이하) → 주황색
- 마감 → 회색
- 방산/이중용도 → 연한 적색 외곽선
- 인접/보조 카테고리 → 보조 회색

**마감일 표시 규칙**:
- 활성: `D-N` 강조
- 마감 후: 회색 처리 + "마감 (n일 경과)"
- 마감일 출처(`deadlineSource`) 호버 시 툴팁: "HTML 본문 / 첨부 PDF / 첨부 HWP" 등 근거 안내

## 6. 필터 패널 항목 (Spec)

| 필터 | 형태 | API 파라미터 | 비고 |
|---|---|---|---|
| 신청자 유형 | 라디오 | `applicantType` | company / startup_or_prefounder / **researcher_or_lab** (기본) |
| 출처 패밀리 | 다중 체크 | `sourceFamilies` | KARI/KASA/KASI/DAPA 등 12종 |
| 우주 카테고리 | 다중 체크 | `spaceCategories` | satellite / launch / defense_space / aerospace_adjacent 등 |
| 마감 윈도우 | 숫자 입력 | `deadlineWithinDays` | 일 단위 (예: 30/60/90) |
| 마감 포함 | 체크박스 | `includeClosed` | 기본 false |
| 방산/이중용도 포함 | 체크박스 | `includeDefense` | 기본 true (연구자 페르소나) |
| 인접 카테고리 포함 | 체크박스 | `includeAdjacent` | 기본 true |
| 컨소시엄 필수 | 3-state | `consortiumRequired` | true / false / any |
| 지역 | 다중 체크 | `regions` | 전국 + 광역 17개 |
| 정렬 | 셀렉트 | `sortBy` | relevance / deadline / announcement_date |

→ 모두 API에 이미 존재. 신규 백엔드 작업 없음.

## 7. 데이터 흐름

```
[브라우저]
   │
   │ GET /space-programs/search?<query>
   ▼
[Next.js (apps/web)]  ──── client fetch ────▶ [apps/api Fastify]  ──▶ generated.json
   ▲                                                    │
   │                                                    ▼
   └── 응답을 카드 렌더링                          /space-programs/:id
```

- 개발 환경: `NEXT_PUBLIC_API_BASE_URL=http://localhost:4000`
- 운영 환경(추후): 같은 도메인 reverse-proxy 또는 Vercel + 별도 API 호스팅 결정
- 검색 입력 디바운스: 350ms
- 응답 캐싱: SWR (`stale-while-revalidate`) 패턴, 30초

에러 처리:
- API 다운: 정적 폴백 메시지 "API 연결 실패. 잠시 후 다시 시도해주세요."
- 빈 결과: "선택한 조건의 활성 공고가 없습니다. 필터를 완화해 보세요." + 활성 필터 칩 보여주기

## 9. 비기능 요구

- **접근성**: 키보드 탭 순서, 필터 라벨 명시, 카드에 aria-label, 색만으로 정보 전달 금지.
- **반응형**: 1280px / 1024px / 768px / 360px 4단계. 모바일은 필터 패널이 슬라이드-인 시트.
- **성능**: 첫 로드 < 1.5s (LCP), API 응답 < 200ms 목표 (로컬).
- **i18n 준비**: 한국어 메인, 추후 영어 토글 자리만 마련 (`/en` 라우트는 v2).

## 10. 구현 단계 (Phase)

| Phase | 범위 | 산출물 | 예상 작업량 |
|---|---|---|---|
| **P1 골격** | `/` 페이지에 탭 3종 라우트 + "필터 검색" 탭만 동작 + 결과 카드 + API 호출 | 검색 가능한 MVP | 중 |
| **P2 상세** | `/program/[id]` 상세 페이지 + 마감 근거/원기관 링크 노출 | 깊이 있는 정보 | 소 |
| **P3 키워드 탭** | 자유 텍스트 검색 + 디바운스 + 관련도 정렬 | 자유 검색 | 소 |
| **P4 매칭 탭** | 연구실 프로필 폼 → fit-score 정렬 + 위험요소/누락 정보 표시 | 추천 모드 | 중 |
| **P5 폴리시** | 빈 상태/에러/로딩 스켈레톤/About 페이지/면책 | 마감 처리 | 소 |
| **P6 디플로이** | Vercel 또는 자체 호스팅 결정 + 환경 분리 | 운영 진입 | 변동 |

P1 만으로도 의미 있는 첫 릴리스. 이후 단계적으로 P2~P5.
